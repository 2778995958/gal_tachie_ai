# TLGmux 立繪分塊格式與 batch_decode 解碼器

TLGmux 是 Kirikiri 引擎用於**立繪（立ち絵）**的分塊圖片格式。一張大型角色立繪被水平切割成多個 tile，每個 tile 獨立用 TLGqoi 編碼，再打包在一個 TLGmux 容器中。

---

## 1. 為什麼要分塊？

立繪通常很高很窄（如 1632×4449），但大部分區域是透明的：

```
ひよ里a_981.tlg: 1632×4449
┌──────────────────┐ ← tile 0: 只有 699px 寬（其餘透明）
│      ████        │
│     ██████       │ ← tile 1: 770px 寬
│    ████████      │
│    ████████      │ ← tile 3: 832px 寬
│    █████████     │
│    ████████      │
│   ██████████████ │ ← tile 8: 1613px 寬（最寬，腰部）
│  ███████████████ │ ← tile 9: 1618px 寬
│     ████████     │
│      ██████      │
│      ██████      │ ← tile 15~16: 腳部，較窄
│      ██████      │
└──────────────────┘
```

每個 tile 只儲存非透明的區域，記錄 `(x, y, w, h)` 座標。透明像素完全不編碼，大幅節省空間。

---

## 2. TLGmux 檔案結構

以 `ひよ里a_981.tlg`（2,992,655 bytes）為例：

```
┌──────────────────────────────────────────────────┐
│ File Header (20 bytes)                           │
│   "TLGmux\x00idx\x1a"  (11 bytes) 魔術數字       │
│   channels = 4          (1 byte)                  │
│   width = 1632          (4 bytes, uint32 LE)      │
│   height = 4449         (4 bytes, uint32 LE)      │
├──────────────────────────────────────────────────┤
│ CMUX Tag (8 bytes)                               │
│   "CMUX"                (4 bytes) 標籤            │
│   cmux_data_size = 412  (4 bytes)                 │
├──────────────────────────────────────────────────┤
│ CMUX Data (412 bytes)                            │
│   entry_count = 17      (4 bytes) tile 數量       │
│   entry[0] (24 bytes)   第 1 個 tile 的資訊       │
│   entry[1] (24 bytes)                             │
│   ...                                             │
│   entry[16] (24 bytes)  第 17 個 tile 的資訊      │
├──────────────────────────────────────────────────┤
│ 8 bytes padding (全零)                            │
├──────────────────────────────────────────────────┤
│ Tile Data                                        │
│   tile[0]: TLGqoi single-image (157,821 bytes)   │
│   tile[1]: TLGqoi single-image (303,636 bytes)   │
│   ...                                             │
│   tile[16]: TLGqoi single-image                   │
└──────────────────────────────────────────────────┘
```

### 與 CG 容器的關鍵差異

| | TLGmux (立繪) | TLGqoi+QHDR (CG 容器) |
|---|---|---|
| 魔術數字 | `TLGmux\x00idx\x1a` | `TLGqoi\x00raw\x1a` |
| 索引標籤 | CMUX | QHDR |
| tile 內容 | 各 tile 尺寸不同 | 所有圖片尺寸相同 |
| 壓縮方式 | 每個 tile 獨立 QOI 編碼 | 所有圖片交錯後統一 QOI+dist 編碼 |
| 用途 | 一張大圖的空間裁剪 | 多張相似圖的壓縮 |
| 輸出 | 1 張完整圖（tile 拼合） | N 張獨立圖 |

## 3. CMUX Entry 結構 (每個 24 bytes)

```
Offset  Size  Description
──────  ────  ──────────────────────────
0x00    4     x（tile 左上角 X 座標）
0x04    4     y（tile 左上角 Y 座標）
0x08    4     w（tile 寬度）
0x0C    4     h（tile 高度）
0x10    4     tile_offset（tile data 的位元組偏移，相對於 tile data 起始）
0x14    4     (保留，通常為 0)
```

ひよ里a_981 的 17 個 tile：

```
Index  Position     Size       Offset      說明
─────  ──────────   ────────   ─────────── ──────
  0    (465, 0)     699×262    0           頭頂
  1    (421, 262)   770×262    157,821     頭部
  2    (376, 524)   898×262    461,457     肩膀
  3    (419, 786)   832×262    720,984     上身
  4    (437, 1048)  841×262    927,997     胸部
  5    (452, 1310)  746×262    1,178,658   腰部上
  6    (370, 1572)  872×262    1,392,704   腰部
  7    (191, 1834)  1257×262   1,553,303   手臂
  8    (19, 2096)   1613×262   1,813,922   手/腰（最寬）
  9    (0, 2358)    1618×262   2,043,702   臀部（最寬）
  10   (260, 2620)  1061×262   2,346,560   大腿上
  11   (495, 2882)  660×262    2,529,909   大腿
  12   (509, 3144)  667×262    2,638,842   膝蓋
  13   (513, 3406)  664×262    2,723,882   小腿上
  14   (548, 3668)  619×262    2,784,551   小腿
  15   (589, 3930)  579×262    2,839,402   腳踝
  16   (588, 4192)  580×257    2,904,217   腳（最後一塊，h=257）
```

觀察：
- 所有 tile 高度固定為 262（最後一塊 257 = 4449 mod 262）
- 寬度隨角色輪廓變化（頭部窄、腰部寬、腿部窄）
- X 座標表示該 tile 的水平偏移（角色不一定置中）

## 4. Tile Data 格式

每個 tile 是一個獨立的 **單張 TLGqoi**（與 decode_single.py 處理的格式相同）：

```
tile data 起始位置 = 28 + cmux_data_size + 8（8 bytes padding）

每個 tile：
  "TLGqoi\x00raw\x1a"  (11 bytes) 魔術數字
  channels              (1 byte)
  tile_width            (4 bytes)
  tile_height           (4 bytes)
  [8 bytes]             (padding/unknown)
  QOI stream            (直到下一個 tile 或檔案結尾)
```

tile_offset（CMUX entry 中的偏移值）是相對於 tile data 起始位置的位元組偏移。

---

## 5. 解碼流程

```
1. 讀取 File Header
   → canvas_width=1632, canvas_height=4449

2. 讀取 CMUX
   → 17 個 tile entries: [(x, y, w, h, offset), ...]

3. 建立空白畫布
   → RGBA canvas (1632×4449), 全透明

4. 計算 tile data 起始位置
   → base = 28 + cmux_size + 8

5. 對每個 tile：
   a. 定位 tile 資料: tile_start = base + tile_offset
   b. 驗證 TLGqoi 魔術數字
   c. 用 QOI 解碼器解碼 tile (w×h 像素)
   d. 將 tile 貼到畫布的 (x, y) 位置

6. 輸出 PNG
```

## 7. 壓縮效率

以 ひよ里a_981 為例：

```
完整畫布：1632 × 4449 × 4 = 29,045,952 bytes (27.7 MB)
TLGmux 檔案：2,992,655 bytes (2.85 MB)
壓縮比：9.7:1

效率來源：
  1. 分塊裁剪：只編碼非透明區域
     實際編碼像素 = Σ(tile_w × tile_h) = ~4,000,000
     vs 完整畫布 7,261,488 像素
     節省 ~45% 像素

  2. QOI 編碼：在非透明區域內進一步壓縮
     實際編碼像素 × 4 = ~16 MB → 2.85 MB
     QOI 壓縮比 ≈ 5.6:1
```
